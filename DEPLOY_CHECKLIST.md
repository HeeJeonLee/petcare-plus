## 🚀 배포 단계 (5분)

### ⏱️ 0:00 - 0:30 | Vercel 계정 만들기

```
□ 1. https://vercel.com 접속
□ 2. "Sign Up" 클릭
□ 3. "Continue with GitHub" 선택
□ 4. GitHub 로그인
□ 5. Vercel 권한 승인
```

**GitHub 계정 없으면:**
```
□ 1. https://github.com 접속
□ 2. "Sign Up" 클릭
□ 3. 이메일/비밀번호 입력
□ 4. 인증 완료
□ 5. 다시 Vercel로!
```

---

### ⏱️ 0:30 - 1:30 | 프로젝트 생성

```
□ 1. Vercel 대시보드에서 "Add New..." 클릭
□ 2. "Project" 선택
□ 3. "Continue with 템플릿" 건너뛰기
□ 4. "Browse" 또는 "Upload" 클릭
```

---

### ⏱️ 1:30 - 3:00 | 파일 업로드

**중요: 모든 파일을 함께 선택!**

```
□ 1. 파일 탐색기 열기
□ 2. petcare-deploy 폴더 찾기
□ 3. Ctrl+A (전체 선택) 또는:
     - index.html 클릭
     - Shift 누른 채로 vercel.json 클릭
□ 4. 선택한 파일들을 Vercel 창으로 드래그
□ 5. 파일 4개가 모두 올라갔는지 확인:
     ✓ index.html
     ✓ signup.html
     ✓ vercel.json
     ✓ README.md
```

---

### ⏱️ 3:00 - 3:30 | 프로젝트 설정

```
□ 1. Project Name: "petcare-plus" 입력
     (또는 원하는 이름)
□ 2. Framework Preset: "Other" 선택
     (자동 선택됨)
□ 3. Root Directory: 그냥 "." 그대로
□ 4. Build Command: 비워두기 (HTML이라 불필요)
□ 5. Output Directory: 비워두기
```

---

### ⏱️ 3:30 - 4:00 | 배포!

```
□ 1. "Deploy" 버튼 클릭!
□ 2. 30초~1분 대기
     → "Building..." 표시됨
     → "Deploying..." 표시됨
     → "Success!" 🎉
```

## 🆘 문제 발생 시

### "파일 업로드 안 됨"
```
→ 파일 크기 확인 (100MB 이하여야 함)
→ 인터넷 연결 확인
→ 브라우저 새로고침 후 재시도
```

### "Build Failed"
```
→ 괜찮습니다! HTML은 빌드 불필요
→ 무시하고 진행
→ 그래도 안 되면 Framework를 "Other"로 변경
```

### "404 Not Found"
```
→ vercel.json 파일이 업로드되었는지 확인
→ 프로젝트 재배포:
  Settings → Redeploy
```

### "디자인이 깨짐"
```
→ 브라우저 캐시 삭제 (Ctrl+Shift+R)
→ 시크릿 모드로 접속
→ 다른 브라우저로 시도
```
